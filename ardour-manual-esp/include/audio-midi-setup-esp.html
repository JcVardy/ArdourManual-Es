Configuración de Audio/MIDI

Esta ventana muestra las diferentes opciones de audio que Ardour usará para la sesión de trabajo actual, tanto para hardware como para software, y está compuesta por:

Sistema de Audio Dependiendo del sistema operativo, Ardour puede utilizar diferentes sistemas de audio. Por ejemplo, en Linux, se pueden usar tanto ALSA, PulseAudio y JACK. En Mac OS X, normalmente se utilizará CoreAudio. Los usuarios avanzados en todas las plataformas también pueden utilizar NetJack, que proporciona E/S de audio a través de red.
Dispositivo de Entrada El selector debe mostrar todas las interfaces disponibles proporcionadas por el sistema de audio anterior y que sean capaces de capturar audio.
Dispositivo de Salida El selector debe mostrar todas las interfaces disponibles proporcionadas por el sistema de audio anterior y que sean capaces de reproducir audio.
Tasa de Muestreo El selector permitirá seleccionar cualquier tasa de muestreo soportada por el dispositivo seleccionado anteriormente.
Tamaño de Búfer El tamaño del búfer utilizado por la interfaz de audio se puede ajustar para permitir una menor latencia, o un menor uso de CPU y mayor latencia.
Periodos Número de cuadros entre cada interrupción de hardware (conversión análogo-digital o digital-análogo).
Monitoreo de Hardware Esta sección permite elegir si Ardour o el hardware de audio debe manejar el monitoreo de grabación.
Ajustes Avanzados Esta sección contiene varios ajustes de bajo nivel listados a continuación.
Sistema MIDI Esto permite elegir un controlador MIDI. En macOS, esto será CoreMIDI. En Linux, la elección será entre el secuenciador ALSA y los dispositivos crudos ALSA. Elegir "Ninguno" desactivará la conexión al backend MIDI externo y evitará que Ardour sea expuesto a puertos MIDI externos, mientras permite enrutar eventos MIDI dentro del programa.
Configuración y Calibración Este botón ejecuta un proceso guiado semi-automatizado para obtener mediciones precisas de latencia de hardware para los puertos MIDI disponibles.
Latencia de Entrada/Salida de Hardware Especificar el retraso de hardware en muestras para una compensación precisa de latencia.
Calibrar Audio Este botón ejecuta un proceso guiado semi-automatizado para obtener mediciones precisas de latencia de hardware para los puertos de audio disponibles.
