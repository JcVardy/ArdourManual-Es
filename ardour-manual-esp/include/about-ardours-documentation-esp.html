Convenciones utilizadas en este manual

Esta sección cubre algunas de las convenciones tipográficas y de lenguaje utilizadas en este manual.

Teclados y Modificadores

Las asignaciones de teclado se muestran así: s o x.

x significa "presiona la tecla , mantenla presionada y luego también presiona la tecla x".

Se pueden ver combinaciones como e, lo que significa "mantén presionada la tecla y la tecla , y luego, mientras mantienes ambas presionadas, pulsa la tecla e".

Las diferentes plataformas tienen distintas convenciones sobre qué tecla modificadora (Control o Command) usar como la principal o más común. Cuando se visualiza este manual desde una máquina que se identifica como ejecutando OS X, se verá Cmd donde corresponda (por ejemplo, en el primer ejemplo de arriba). En otras máquinas, se verá Ctrl en su lugar.

Botones del Ratón

Los botones del ratón se mencionan como Izquierdo, Central y Derecho. Ardour puede utilizar botones adicionales, pero no tienen un comportamiento predeterminado en el programa.

Modificadores de Clic del Ratón

Muchas funciones de edición se realizan haciendo clic con el ratón mientras se mantiene presionada una tecla modificadora, por ejemplo, Izquierdo.

Rueda del Ratón

Algunos elementos de la interfaz gráfica pueden controlarse opcionalmente con la rueda del ratón cuando el puntero se encuentra sobre ellos. La notación para la acción de la rueda del ratón es ⇑ ⇐ ⇓ ⇒.

Clic de Contexto

El término clic de contexto se usa para indicar un clic derecho sobre un elemento específico de la interfaz gráfica. Aunque el clic derecho es la forma común y predeterminada de hacerlo, existen otras maneras de lograr lo mismo—este término se refiere a cualquiera de ellas, y el resultado siempre será la aparición de un menú específico para el elemento seleccionado.

"El Puntero"

Cuando el manual se refiere al "puntero", significa la representación en pantalla de la posición del ratón o la ubicación de una acción táctil si se está utilizando una interfaz táctil.

Otras Formas de Entrada del Usuario

Ardour admite controladores de hardware, como bancos de deslizadores (faders), perillas o botones.

Elementos del Menú

Los elementos del menú se indican de la siguiente manera: Superior > Siguiente > Más Profundo. Cada elemento separado por ">" indica un nivel dentro de un menú anidado o submenú.

Mensajes OSC

Los mensajes OSC, ya sean enviados o recibidos, se muestran así: /transport_stop.

Opciones de Preferencias/Diálogo

Las opciones en varios cuadros de diálogo, especialmente en las Preferencias y Propiedades, se indican de la siguiente manera:

Editar > Preferencias > Audio > Alguna opción.

Cada elemento sucesivo indica un menú, submenú o una pestaña de navegación en un diálogo. El último elemento es el que se debe elegir o seleccionar.

Si una opción está deseleccionada, se verá así:

Editar > Preferencias > Audio > Alguna otra ppción.

Entradas del Usuario

Algunos diálogos o funciones pueden requerir que el usuario ingrese datos como este. En casos raros, ciertas operaciones deberán realizarse en la línea de comandos del sistema operativo:

cat /proc/cpuinfo sleep 3600 ping www.google.com

Salida del Programa

Los mensajes importantes de Ardour u otros programas se mostrarán de esta manera.

Notas

Notas importantes sobre aspectos que podrían no ser obvios se muestran en este formato.

Advertencias

Problemas serios que podrían causar fallos, pérdida de datos, afectar la calidad del sonido o devorar metafóricamente a tu pez dorado, se muestran de esta manera.
